# 057 — Optimize IC14 (weighted shortest path) [chickpeas]

chickpeas 14 ms / Kùzu 13 ms (the only IC loss).

chickpeas uses the unidirectional single-source `dijkstra` + a per-edge
interaction-map HashMap lookup. Measured breakdown: bidirectional
`weighted_shortest_path` ~7 ms (would beat Kùzu); unit-weight dijkstra ~5 ms
(the hash lookup costs ~9 ms). Options: switch IC14 to the bidirectional
primitive (cost only; lose the path), and/or precompute a per-edge weight to
drop the hash lookup. Decide whether IC14 must still return a path.

Timings are SF1 under load avg ~6.7 (read relatively; re-measure on a quiet machine). Acceptance: faster on the targeted engine AND still byte-identical via kuzu/compare.py.
